# YoptaScript Dictionary / Словарь YoptaScript

Словарь соответствий между ключевыми словами JavaScript и их аналогами в YoptaScript.
Маппинг выровнен по оригинальному [samgozman/YoptaScript](https://github.com/samgozman/YoptaScript);
расхождения и намеренно неподдержанные элементы перечислены в конце.

> Источник истины — лексер (`crates/yps-lexer/src/lexer.rs`) и список глобалов
> (`crates/yps-interpreter/src/stdlib/mod.rs`). Эта таблица отражает фактическое поведение.

## Намеренно НЕ поддерживается

Эти элементы есть в словаре оригинала, но в `yoptascript-rs` сознательно не реализованы:
наш проект — **интерпретатор языка**, а не браузерный JS-транспайлер, поэтому браузерные
и Java-наследованные сущности без рантайм-смысла в JS опущены.

- **`}` → `есть`.** Слово `есть` уже занято нашей stdlib (`Отражение.есть` = `Reflect.has`,
  proxy-ловушка `есть`). Сделать его ключевым словом `}` сломало бы рабочий API.
  Используйте литерал `}`. Пара `жЫ` = `{` сохранена.
- **DOM `document.*`** — сотни браузерных методов/свойств/обработчиков событий
  (`ксива.малява`, `вычислитьЛохаПоНомеру` и т.п.). У интерпретатора нет DOM.
- **Java-наследованные ключевые слова** без смысла в JS-рантайме:
  `abstract`/`говнойбать`, `final`/`бачок`, `implements`/`силикон`, `interface`/`хуёво`,
  `synchronized`/`вписон`, `native`/`чорт`, `transient`/`ахз`, `volatile`/`вписос`,
  `throws`/`плюнуть`, `package`/`клеёнка`, `enum`/`еээ`, `goto`/`пиздуй`, `with`/`хзйопт`,
  `eval`/`ебал`.
- **Java-типы** `Float`/`Плавник`, `Double`/`Двойные`, `Long`/`Колонна`, `Short`/`Пипин`,
  `Byte`/`Семка`, `Char`/`Эээ`, `Boolean`/`Пацан` — в JS нет отдельных примитивных
  типов такого рода.
